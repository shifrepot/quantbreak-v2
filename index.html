02 · 양자 구조가 하는 역할
Rᵖ(Σ) Hilbert Space
내적 ⟨u,v⟩_Σ = uᵀΣv
GBM은 표준 유클리드 공간에서 단일 변수(σ)로 위험을 측정합니다. 우리는 시장 공분산 행렬 Σ로 정의된 Hilbert 공간 Rᵖ(Σ)에 5개 시장변수를 투영합니다. 이 공간에서 내적은 ⟨u,v⟩_Σ = uᵀΣv로 정의되며, SIR은 이 공간 안에서 공분산 구조를 반영한 저차원 위험 방향 β₁을 추출합니다.
Density Matrix ρ
시장 레짐 혼합 상태 (Mixed State)
GBM은 레짐 전환을 명시적으로 모델링하지 않습니다. Density Matrix는 시장을 CRASH / ELEVATED / SAFE 세 레짐의 확률적 혼합 상태(mixed state)로 표현합니다. 각 레짐은 Hilbert Space의 직교 기저로 정의되며, 대각 원소는 SIR projection에 GaussianHMM(3 states)을 적합하여 추정한 레짐 확률입니다. 분위수 균등분할(항상 33:33:34)과 달리, HMM은 시장이 실제로 어느 상태에 머물렀는지를 데이터에서 학습하므로 p_crash가 시장 상황에 따라 실제로 달라집니다. 본 연구에서는 레짐 간 coherence는 가정하지 않습니다.

ρ = p_crash|crash⟩⟨crash| + p_elev|elev⟩⟨elev| + p_safe|safe⟩⟨safe|
Born Rule 측정 (Measurement)
P(i) = Tr(Πᵢ·ρ) = pᵢ
Born Rule의 measurement postulate P(i) = Tr(Πᵢ·ρ)를 통해 각 레짐의 측정 확률을 얻습니다. 대각 ρ에서 이는 대각 원소를 읽는 것과 동일합니다. 측정된 확률(p_crash, p_elev, p_safe)을 H_risk = diag(CVaR)의 기대값 계산에 사용하여 Tr(ρH_risk) = Σᵢ pᵢ·CVaRᵢ로 이어집니다.
QAOA 조합 최적화
12 qubits · 81 combinations
4개 자산 × 3개 노출 레벨 = 12개 이진변수. QAOA는 균등 중첩에서 시작해 Phase kickback(비용 Hamiltonian)과 RX Mixer를 2층 반복해 낮은 비용 상태로 수렴합니다. 단순히 위험만 최소화하면 항상 Low만 선택되는 단조 문제가 생기므로, 기대수익 Reward 항을 빼서 진짜 트레이드오프를 만들었습니다.
QAOA 회로 구조 (12 qubits, p=2)
q₀ ─[H]─[e^{-iγH_C}]─[RX(2β)]─[e^{-iγH_C}]─[RX(2β)]─[M]
q₁ ─[H]─[e^{-iγH_C}]─[RX(2β)]─[e^{-iγH_C}]─[RX(2β)]─[M]
⋮                                                                                  ⋮
q₁₁─[H]─[e^{-iγH_C}]─[RX(2β)]─[e^{-iγH_C}]─[RX(2β)]─[M]

Why SIR? — Finding the True Risk Direction

The relationship between market variables X and future tail loss Y = f(X) is nonlinear. GBM ignores this nonlinearity. SIR (Sliced Inverse Regression) finds the direction β that best explains Y without assuming a functional form for f — making it the right tool for tail risk analysis.

Y = f(X)                                   // true relationship is unknown and nonlinear (Y = 20-day forward drawdown)
M = Σₕ wₕ(m̄ₕ − m̄)(m̄ₕ − m̄)ᵀ          // SIR: between-slice covariance over Y-slices
β = eigenvec(Σ⁻¹M)                              // SDR direction recovered without assuming f's form

MODULE C · REGIME-WEIGHTED RISK SCORE

Using regime probabilities from Born Rule above, we compute a Regime-Weighted Risk Score — Tr(ρH_risk) = Σᵢ pᵢ·CVaRᵢ, the HMM-weighted average of per-regime Historical CVaRs. This is not claimed to be more accurate than a flat Historical CVaR. Rather, it reflects which regime the market is currently in — if HMM assigns high p_crash, the score skews toward crash-regime CVaR; if p_safe dominates, it skews toward the safe-regime CVaR. The score feeds into QAOA as a regime-aware signal. GBM vs Historical CVaR comparison is shown in Tab ②.

④ QAOA Multi-Asset Risk Allocation

A genuine combinatorial optimization: 4 assets × 3 exposure levels = 12 binary variables, 3⁴=81 combinations. QAOA balances Tr(ρ·H_risk) (tail risk) against Tr(ρ·H_return) (expected return) per asset — minimizing risk alone always picks the lowest exposure for everything, so the objective must be risk-adjusted, not risk-only.

Note: this tab always recomputes SDR → Density Matrix → Tr(ρH_risk) from scratch for a fixed set of 4 assets (TQQQ, SOXL, WTI, NG) — it does not reuse whatever you ran in Tab ③. Tab ③ is a single-asset exploration tool; Tab ④ solves an independent 4-asset allocation problem. Running this button re-fetches live market data and re-runs the full pipeline.

PER-ASSET REGIME RISK — SDR + DENSITY MATRIX + BORN RULE

Each asset gets its own SIR projection (Y=20-day forward drawdown), regime probabilities via Born Rule, and Tr(ρ·H_risk) — the expected tail risk under the current regime mixture.

Numpy backend: full 12-qubit QUBO (81 combinations), statevector simulation via vectorized tensor ops.
AerSimulator backend: real Qiskit quantum circuit (H, diagonal cost unitary, RX mixer, measurement) on a reduced 4-qubit (2 assets × 2 levels) version — verifies that an actual circuit converges to the same solution as the numpy statevector. The full 12-qubit circuit would require a 4096×4096 dense unitary that exceeds serverless timeout limits, so this is an honest scope reduction, not a shortcut.

WHY QAOA HERE?
Single-asset leverage selection is a continuous linear scale — always trivially solved (lowest leverage minimizes risk-only cost). Multi-asset risk allocation is genuinely combinatorial: 4 assets × 3 levels = 12 binary variables with one-hot constraints per asset, 81 valid combinations. The cost function also balances tail risk against expected return (risk-only minimization would again trivially pick the lowest exposure for every asset) — this risk-adjusted, multi-asset structure is what makes the QUBO non-trivial. As assets and exposure tiers grow (10 assets × 4 levels = 4¹⁰ ≈ 1M), classical exhaustive search becomes intractable — this is exactly the structure QAOA is designed for.

This project does not claim quantum speedup at this scale. The numpy backend solves the full 12-qubit problem; the AerSimulator backend verifies a reduced 4-qubit version on a real quantum circuit. The objective is to show GBM's limits → SDR → Density Matrix → Born Rule → Risk Hamiltonian → QUBO → QAOA as one coherent pipeline.

📖 용어 사전

이 앱에서 사용되는 주요 용어를 쉽게 정리했습니다.

금융 · 리스크
GBM
Geometric Brownian Motion. 주가가 정규분포를 따라 움직인다고 가정하는 전통적 금융 모델. Black-Scholes 옵션 공식의 기반이 되는 모형이다.
CVaR
Conditional Value at Risk. 가장 나쁜 5% 상황에서의 평균 손실. "최악의 상황에서 얼마나 잃는가"를 나타내는 리스크 지표. Expected Shortfall(ES)이라고도 한다.
Fat Tail
정규분포보다 극단적 사건이 더 자주 발생하는 분포의 특성. 금융 시장에서 대형 폭락이 GBM 예측보다 훨씬 자주 일어나는 현상을 설명한다.
Excess Kurtosis
분포의 꼬리 두께를 나타내는 통계량. 정규분포는 0. 0보다 크면 Fat Tail(극단값이 정규분포보다 자주 발생), 0보다 작으면 Thin Tail이다.
Volatility Decay
레버리지 ETF에서 매일 리밸런싱으로 인해 변동성이 쌓일수록 수익률이 비선형적으로 감소하는 현상. 3배 레버리지가 장기적으로 기초자산의 3배 수익을 내지 못하는 이유.
Regime (레짐)
시장의 상태 구간. 이 앱에서는 CRASH(위기), ELEVATED(위험 상승), SAFE(안정)의 세 레짐으로 분류한다. 레짐마다 자산의 위험과 수익 특성이 다르다.
Tail Event Ratio
실제 극단적 손실 발생 빈도를 GBM 예측 확률로 나눈 값. 8.3×이면 GBM이 예측한 것보다 8.3배 더 자주 극단적 손실이 발생했다는 뜻이다.
GBM Blind Spot
GBM이 안전하다고 분류하지만 실제로는 위험한 구간. Volatility Decay 차트에서 실제 3배 레버리지 경로가 GBM CVaR 수준을 처음 돌파하는 시점이 Blind Spot의 시작이다.
통계 방법론
SDR
Sufficient Dimension Reduction. 고차원 데이터를 목표 변수(Y)와의 관계를 유지하면서 저차원으로 줄이는 통계 방법론의 총칭. SIR이 SDR의 대표적 구현 방법이다.
SIR
Sliced Inverse Regression (Li, 1991). 5개 시장변수를 "미래 위험(Y)을 가장 잘 설명하는 1개 방향(β₁)"으로 압축하는 알고리즘. Y를 구간으로 나눠 각 구간의 X 평균에서 방향을 추출한다.
β₁ (베타1)
SIR이 찾아낸 1번째 위험 방향 벡터. 5차원 시장변수(VIX·SPY·TLT·GLD·금리)를 이 방향으로 투영하면 미래 위험과 가장 상관관계가 높은 1차원 값이 나온다.
Hilbert Space Rᵖ(Σ)
시장 공분산 행렬 Σ로 내적을 정의한 수학적 공간. ⟨u,v⟩ = uᵀΣv. 변수 간 상관관계를 반영한 "거리"를 사용해 SIR이 위험 방향을 추출하는 공간이다.
Forward Drawdown
특정 시점 이후 일정 기간 동안 발생한 최대 낙폭. 이 앱에서 Y는 20일 forward drawdown, 즉 "오늘 이후 20 거래일 안에 발생할 수 있는 최악의 손실"로 정의된다.
양자 개념
Density Matrix ρ
양자 시스템의 상태를 확률적으로 표현하는 행렬. 이 앱에서는 시장이 CRASH / ELEVATED / SAFE 레짐에 각각 어떤 확률로 있는지를 행렬로 표현한다. 대각 원소 = 레짐 확률.
Mixed State
여러 상태가 확률적으로 섞인 상태. GBM이 "시장은 하나의 상태"라고 보는 것과 달리, 이 앱은 시장이 CRASH·ELEVATED·SAFE 레짐의 혼합 상태에 있다고 본다.
Born Rule
양자 측정의 기본 원리. 어떤 상태를 측정했을 때 특정 결과가 나올 확률은 P(i) = Tr(Πᵢ·ρ). 이 앱에서는 Density Matrix에서 레짐 확률을 읽는 과정에 해당한다.
Hamiltonian
양자역학에서 시스템의 에너지를 나타내는 연산자. 이 앱의 Risk Hamiltonian H_risk = diag(CVaR)는 각 레짐의 꼬리위험을 대각 원소로 갖는 행렬이다.
Tr(ρH_risk)
Density Matrix와 Risk Hamiltonian의 trace(대각합). 레짐 확률로 가중평균한 기대 꼬리위험. Σᵢ pᵢ·CVaRᵢ와 동일하며 QAOA 비용함수의 핵심 입력값이다.
Bra-Ket 표기
양자역학의 벡터 표기법. |crash⟩는 "CRASH 레짐 상태 벡터", ⟨crash|는 그 켤레 벡터. |crash⟩⟨crash|는 두 벡터의 외적으로 Density Matrix의 각 항을 구성한다.
Qubit
양자 컴퓨터의 기본 정보 단위. 고전 컴퓨터의 bit(0 또는 1)와 달리 0과 1의 중첩 상태를 가질 수 있다. 이 앱에서는 12 qubit로 81개 포트폴리오 조합을 동시에 탐색한다.
Hadamard Gate
모든 가능한 상태를 동일한 확률로 중첩시키는 양자 게이트. QAOA 시작 시 12개 qubit에 각각 적용해 4096개 포트폴리오 조합 모두를 동시에 탐색하는 초기 상태를 만든다.
Phase Kickback
QAOA에서 비용 Hamiltonian을 적용하는 과정. 각 포트폴리오 조합에 비용에 비례한 위상을 부여한다. 비용이 낮은 조합과 높은 조합의 위상이 달라져 이후 Mixer를 통한 간섭이 가능해진다.
최적화
QUBO
Quadratic Unconstrained Binary Optimization. 이진변수(0 또는 1)로 구성된 이차 목적함수를 최소화하는 문제. 양자 컴퓨터가 native하게 풀 수 있는 형식이다.
QAOA
Quantum Approximate Optimization Algorithm. QUBO를 양자 변분 회로로 근사 최적해를 찾는 알고리즘. Phase kickback(비용 적용)과 Mixer(탐색)를 반복해 낮은 비용 상태의 확률을 높인다.
One-Hot 인코딩
여러 선택지 중 하나만 1로 표시하는 이진 인코딩 방식. High/Medium/Low의 삼진 선택을 [1,0,0], [0,1,0], [0,0,1] 형태의 이진벡터로 변환해 QUBO에 사용한다.
변분 알고리즘
파라미터(γ, β)를 바꿔가며 목적함수를 최소화하는 방식. QAOA는 파라미터 조합마다 양자 회로를 실행하고 에너지 기대값을 계산해 최적 파라미터를 찾는다.
Exposure (노출)
특정 자산에 투자하는 비중. High=100%, Medium=50%, Low=15%. QAOA가 각 자산별로 최적 노출 수준을 결정한다.
